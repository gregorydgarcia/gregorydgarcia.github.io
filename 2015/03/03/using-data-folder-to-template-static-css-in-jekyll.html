Using _data folder to template static CSS in Jekyll

Sass variables are a great way to seperate user defined styles when writing tempaltes for Jekyll. But what happens when you are working with a code base that does not use Sass? I recently ran into this issue when tweaking a pure Css template. The developer had come up with a good solution of using Yaml variables in the _config.yml file.

color:
  primary: #65457e
  secondary: #13a7c7
  secondary-dark: #123677
.purple-section{
	background-color: {{ page.color.primary }}
}
.blue-section{
	background-color: {{ page.color.secondary }}
}

This works great. The only issue is that to test out different styles in the _config.yml the server needs to be restarted. While this isn’t a game ender, a major selling point of Jekyll is being able to tweak a site’s styling as the code renders in real time.

Is there a way we could take a similiar approach outside the _config.yml file? Jekyll has a great solution already baked into the program: the _data file. Taken from the Jekyll documents

_data folder is where you can store additional data for Jekyll to use when generating your site … This powerful feature allows you to avoid repetition in your templates and to set site specific options without changing _config.yml.

So now it’s just a matter of moving your style variables to their new home _data/template.yml and updating the css doc.

.purple-section{
	background-color: {{ page.data.template.color.primary }}
}
.blue-section{
	background-color: {{ page.data.template.color.secondary }}
}

This may look verbose but when designing a template the majority of the code editing should take place in the template.yml folder and not the css file. I used this technique to template out my colors, fonts, base padding, and images.
